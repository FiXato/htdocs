openMSX: emulate the MSX standard

openMSX is alpha software and currently only intended for developers.

Features

Here are some highlights of the features of the latest release:

  • support for almost all MSX systems, except MSX-MIDI (turbo R) and some other details
  • support for almost all known cartridge mapper types via autodetection or the included ROM database, including
    • Panasonic FS-CA1 MSX Audio
    • National mapper type used for internal software
    • Panasonic mapper type used for internal software (e.g. MSX-View)
    • ROMs with special headers or which start from basic, like Nausicaa, Pairs, etc.
    • all (?) ROMs with SRAM (Hydlide 2, Xanadu, many Koei games, Daisenryaku, etc.)
    • Cross Blaim
    • R-Type
    • Harry Fox Yuki No Maou-Hen
    • Korean multi-game ROMs: Zemmix 126, Game's 80 and MSX90-collection
  • MegaRAM emulation (non-diskROM types)
  • XML files specify the hardware to be emulated, so you actually emulate specific MSX machines, which of course can be modified
  • various screen renderers available:
    • 640×480 hi-res renderer
    • 320×240 lo-res renderer (with compensation for lower resolution by blending)
    • 640×480 openGL based renderer that makes advantage of hardware acceleration of video cards and has some special effects: scanlines and blurring (TV emulation, see screenshots below) and now also afterglow
  • full interlacing support, real or deinterlaced
  • overscan support
  • PAC emulation (Panasonic SW-M001)
  • MSX-DOS2 support
  • Sunrise IDE interface with harddisk emulation
  • Moonsound support (may not be 100% complete)
  • "Moonblaster Stereo" support: MSX Music on one channel, MSX Audio on the other
  • pixel accurate rendering: horizontal screensplits are rendered almost correctly (Unknown Reality)
  • console with alpha-blended background
  • real floppy disk controller support (WD2793 and National MB8877A fully supported, including formatting); also partial support for TC8566AF and Microsol based FDCs
  • kanji JIS ROM 1 and 2 support
  • 12×12 dot kanji support (National FS-4600)
  • mouse and joystick support (with option to use the keyboard as joystick)
  • experimental JoyNet support (connected to TCP/IP of host OS)
  • tape support via CAS and WAV files (real recorded tape image)
  • `real time timing', using the specially developed EmuTime model
  • option to run at maximum possible speed
  • full screen rendering
  • file and command completion in console
  • (auto) frameskip
  • support for .xsa (compressed) diskimages
  • speed setting (in percent)
  • mouse cursor grabbing, for windowed mode
  • support for Konami's Majutsushi and Synthesizer DAC
  • support for SCC and SCC+ (the latter can be used in SD-Snatcher, Snatcher, expanded (128kB RAM) or mirrored (64kB on both banks) mode)
  • support for SIMPL/Covox (sound via the printer port)
  • more than 40 MSX machine hardware configurations included (including two using C-BIOS; the others need original MSX ROMs)
  • more than 15 MSX extension hardware configurations included (may need the original ROMs)
  • tunable volumes for the individual sound chips, from the console

Here are some highlights of features and improvements that are already in CVS and are likely to be in the next release:

  • automatic .cas to .wav conversion in CassettePlayer (no patched ROM needed anymore to run .cas files!)
  • cassette sounds are now audible
  • tapes can be rewinded
  • tapes can be forced to play (like unplugging the 'remote control' plug on a real machine)
  • Jarek Burcynszki's YM2413 core (to replace the other one)
  • a debug device: write data to special I/O ports and they get in stdout or a file
  • MSX-MIDI (for turbo R), you can redirect MIDI to /dev/midi and get sound that way!
  • joystick emulation for mouse (a feature that the Philips SBC-3810 and Sony MOS-1 and similar mice have)
  • MSX-RS-232 support (8255 UART)
  • support for PCM input (sampling) for TurboR via a .wav file
  • support for gzipped (.gz) ROMs and disk images and all other files
  • preliminary support for using directories as disk images
  • keys for keyjoystick are now configurable
  • MSX keymap is now configurable
  • run time stereo settings: you can now set the channel mode of each (mono) sound device to mono, left or right from the console
  • multiple commands possible in the console, separated with ';'

openMSX IRC channel

On the server irc.freenode.net you can find the channel #openMSX where you can chat about openMSX. Often, you can also find developers on that channel.

openMSX cd

On the MSX fair in Bussum of 2002, a self-booting cd was released that can run on many computers, independent of the OS installed on them. This cd was created to demonstrate openMSX to people who do not have Linux installed or who do not or can not download and build openMSX. More information about it will be added here later.
