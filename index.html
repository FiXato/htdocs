openMSX - the MSX emulator that aims for perfection

Note that openMSX is alpha software, which means that some things work but not all features are implemented yet.

Features

Here are some highlights of the features of the latest release (0.5.2).

Legend:

New!new feature since last release
Unique!unique feature for an MSX emulator (no other MSX emulators have it)
Cool!particularly cool feature
1st!openMSX is the first MSX emulator to have this feature (but it's not unique anymore)

General

  • for Linux, Windows, MacOS X, FreeBSD, NetBSD and OpenBSD systems
  • installer for Windows
  • Cool! Debian packaging, shortly after the release
  • Unique! command interface via pipes: you can control openMSX with any external application that can communicate via pipes with openMSX! An example implementation is available.
  • Cool! openMSX Catapult: the optional GUI for openMSX! Uses the interface mentioned in the previous item.
  • Unique! TCL is the internal scripting language: enhance openMSX with powerful scripts! (Some examples are included, including a New! script with hundreds of game trainers!)
  • Advanced build system
  • manuals online and in distribution
  • more than 75 MSX machine hardware configurations included (including three using C-BIOS; the others need original MSX ROMs).
  • more than 30 MSX extension hardware configurations included (may need original ROMs).
  • 1st! `real time timing', using the specially developed EmuTime model
  • 1st! pixel accurate rendering: horizontal screensplits are rendered almost correctly (Unknown Reality, Verti-demo).
  • Unique! system ROMs are specified and found with their SHA1 sums or file names as a fall back. This means: no need to install all the ROMs in separate directories anymore!
  • MMX and extended-MMX optimisations for rendering
  • Cool! On Screen Display, now still mainly used for LEDs.
  • New! DirectSound sound driver for MS Windows

MSX hardware emulated

  • almost all MSX systems, except for some minor details
  • different CPU clock speeds, including those for the Panasonic MSX2+ machines
  • almost all known cartridge mapper types via autodetection or the included ROM database, including
    • 1st! Panasonic FS-CA1 MSX Audio
    • 1st! National mapper type used for internal software
    • Panasonic FS-A1FM internal mapper, with firmware switch
    • Panasonic mapper type used for internal software (e.g. MSX-View)
    • ROMs with special headers or which start from basic, like Nausicaa, Pairs, etc.
    • all (?) ROMs with SRAM (Hydlide 2, Xanadu, Wizardry, many Koei games, Daisenryaku, etc.)
    • Cross Blaim
    • R-Type
    • Harry Fox Yuki No Maou-Hen
    • Korean multi-game ROMs: Zemmix 126, Game's 80, Zemmix 64 and MSX90-collection
  • Unique! Sunrise GFX9000! (experimental)
  • almost all MSX sound devices:
    • Cool! Moonsound (may not be 100% complete)
    • Konami's Majutsushi and Synthesizer DAC
    • Cool! "Moonblaster Stereo": MSX Music on one channel, MSX Audio on the other
    • SCC and SCC+ (the latter can be used in SD-Snatcher, Snatcher, expanded (128kB RAM) or mirrored (64kB on both banks) mode)
    • Checkmark FM Stereo PAK
    • 1st! SIMPL/Covox (sound via the printer port)
    • Unique! MSX-MIDI (turboR)
    • 1st! PCM output (turboR)
    • Unique! PCM input (sampling) for turbo R via a .wav file
  • many MSX storage (disk/SRAM) systems:
    • MegaRAM and Unique! MegaRAM Disk
    • 1st! PAC (Panasonic SW-M001)
    • 1st! Sony HBI-55 datacartridge
    • MSX-DOS2
    • Unique! Sunrise IDE interface with harddisk
    • floppy disk controller (WD2793 and National MB8877A fully supported, including formatting); also partial support for TC8566AF and Microsol based FDCs
  • Cool! interlacing, real or deinterlaced
  • overscan
  • kanji JIS ROM 1 and 2
  • 1st! MSX Bunsetsu Henkan Jisyo ROM (National FS-SR021/FS-4000/FS-4500/4700F)
  • 1st! 12×12 dot kanji (National FS-4600)
  • mouse and joystick (with option to use the keyboard as joystick)
  • Unique! JoyNet (experimental; connected to TCP/IP of host OS)
  • tape via CAS and WAV files (real recorded tape image); CAS files can be converted to WAV on the fly, cassette sounds are audible, cassettes can be rewinded and forced to play (like unplugging the 'remote control' plug on a real machine)
  • MSX RS-232 (8255 UART)
  • MSX printer support by logging the output to a file
  • joystick emulation for mouse (a feature that the Philips SBC-3810 and Sony MOS-1 and similar mice have)
  • rensha turbo auto fire support
  • turboR pause (button) support
  • Unique! slot expander
  • 1st! Tetris II Special Edition dongle

Emulator extras

  • Unique! XML files specify the hardware to be emulated, so you actually emulate specific MSX machines, which of course can be modified
  • various screen renderers available:
    • 640×480 hi-res renderer, with support for 4 different scalers: hq2x, New! hq2xlite, 2xSaI and scale2x (see animated images below)
    • 320×240 lo-res renderer (with compensation for lower resolution by blending)
    • Cool! 640×480 openGL based renderer that makes advantage of hardware acceleration of video cards and has an extra special effect: afterglow (see also screen shots below)
  • for SDLHi and SDLGL renderers: scanlines and blur for TV emulation (see screen shots below)
  • gamma adjustment setting
  • Unique! console with alpha-blended background and file and command completion
  • option to run at maximum possible speed
  • full screen rendering
  • (auto) frameskip
  • Unique! support for .xsa (compressed) diskimages
  • speed setting (in percent)
  • mouse cursor grabbing, for windowed mode
  • tunable volumes and channel modes for the individual sound chips
  • Unique! debug device: write data to special I/O ports and its get logged to stdout or a file in a format you specify. Very useful if you develop MSX assembly programs in openMSX
  • all read-only input files (ROM images, disk images, background pictures, XML) can be gzipped or zipped and will be transparently decompressed by openMSX. Note that some games require write access to their disk in order to run (for example, Seed of Dragon)
  • 1st! dir-as-disk feature: a directory can be specified instead of a disk image and the contents (up to 720K) will show up as files on the MSX. There is no support for subdirectories (as used by DOS2) yet. Note that this feature is still experimental
  • adjustable keymap (e.g. for Japanese PC keyboards or to use Russian MSX keyboards on a US English PC keyboard)
  • screen shot feature
  • support for IPS patches for ROMs and disks
  • Cool! debug command (including break points): read/write many parts of the MSX system; mainly useful to be used in external debug applications via the stdin command interface
  • Unique! keymatrixdown/up commands to automatically push keys (like pressing CTRL at boot automatically)
  • Unique! console commands to enable demo play with timers
  • New! sound logger
  • New! disk manipulator toolkit; transfer files from the host OS to the MSX disks (disk images and hard disk images with partitions are all supported) and vice versa, create new (hard) disk images and format them, etc.

Here are some highlights of features and improvements that are already in CVS and are likely to be in the next release:

  • compilation works fine in gcc 4.x, which comes default on Mac OS X Tiger
  • flusing of printer port logger
  • implementation of screen 6 border and background stripes
  • implemented TurboR hardware pause and hardware PCM mute
  • don't save settings that still have their default value (more easily upgrading)
  • added second key joystick (old bindings will not work anymore)
  • big sound quality improvement of DAC devices (turboR PCM, Konami DAC)
  • boosted baudrate of internal CAS to WAV conversion
  • fixed some graphical glitches on big endian CPUs (like Mac's), including icon
  • extended debugging commands (a.o. conditional breakpoints, but still with addresses)
  • enabled socket communication, including some new settings to set it up; makes connecting at run time possible
  • removed SDLHi and SDLLo renderers, SDLHi is now just "SDL" and you can get SDLLo by using the new "low" scaler
  • V9990: speed improvements, enabled scalers, scanline and blur for V9990 display
  • added -prefix option for screenshot command, useful in combination with a new guess_title script: you can make screenshots with the title of the game you're running as prefix
  • added saving to cassette, for now only for systems with a Jack Audio Connection Kit running (not Windows)

openMSX IRC channel

On the server irc.freenode.net you can find the channel #openMSX where you can chat about openMSX. Often, you can also find developers on that channel.
