openMSX - the MSX emulator that aims for perfection

Note that openMSX is alpha software, which means that some things work but not all features are implemented yet.

Features

Here are some highlights of the features of the latest release (0.4.0).

Legend:

New!new feature since last release
Unique!unique feature for an MSX emulator (no other MSX emulators have it)
Cool!particularly cool feature
1st!openMSX is the first MSX emulator to have this feature (but it's not unique anymore)

General

  • for Linux, Windows, MacOS X and FreeBSD systems
  • New!installer for Windows
  • command interface via pipes: you can control openMSX with any external application that can communicate via pipes with openMSX! An example implementation is available.
  • New! openMSX Catapult: the optional GUI for openMSX! Uses the interface mentioned in the previous item.
  • New! TCL is the internal scripting language: enhance openMSX with powerful scripts! (Some examples are included.)
  • New! Advanced build system
  • manuals online and in distribution
  • more than 60 MSX machine hardware configurations included (including two using C-BIOS; the others need original MSX ROMs).
  • more than 20 MSX extension hardware configurations included (may need original ROMs).
  • Unique! `real time timing', using the specially developed EmuTime model
  • Unique! pixel accurate rendering: horizontal screensplits are rendered almost correctly (Unknown Reality, Verti-demo).

MSX hardware emulated

  • almost all MSX systems, except for some details
  • New! different CPU clock speeds, including those for the Panasonic MSX2+ machines
  • almost all known cartridge mapper types via autodetection or the included ROM database, including
    • Unique! Panasonic FS-CA1 MSX Audio
    • 1st! National mapper type used for internal software
    • New! Panasonic FS-A1FM internal mapper, with frontswitch
    • Panasonic mapper type used for internal software (e.g. MSX-View)
    • ROMs with special headers or which start from basic, like Nausicaa, Pairs, etc.
    • all (?) ROMs with SRAM (Hydlide 2, Xanadu, Wizardry, many Koei games, Daisenryaku, etc.)
    • Cross Blaim
    • R-Type
    • Harry Fox Yuki No Maou-Hen
    • Korean multi-game ROMs: Zemmix 126, Game's 80, Zemmix 64 and MSX90-collection
  • almost all MSX sound devices:
    • Cool! Moonsound (may not be 100% complete)
    • Konami's Majutsushi and Synthesizer DAC
    • Cool! "Moonblaster Stereo": MSX Music on one channel, MSX Audio on the other
    • SCC and SCC+ (the latter can be used in SD-Snatcher, Snatcher, expanded (128kB RAM) or mirrored (64kB on both banks) mode)
    • Unique! SIMPL/Covox (sound via the printer port)
    • Unique! MSX-MIDI (turboR)
    • Unique! PCM output (turboR)
    • Unique! PCM input (sampling) for turbo R via a .wav file
  • many MSX storage (disk/SRAM) systems:
    • MegaRAM (non-diskROM types)
    • Unique! PAC (Panasonic SW-M001)
    • Unique! Sony HBI-55 datacartridge
    • MSX-DOS2
    • Unique! Sunrise IDE interface with harddisk
    • floppy disk controller (WD2793 and National MB8877A fully supported, including formatting); also partial support for TC8566AF and Microsol based FDCs (improved in latest release)
  • 1st! interlacing, real or deinterlaced
  • overscan
  • kanji JIS ROM 1 and 2
  • 1st! MSX Bunsetsu Henkan Jisyo ROM (National FS-SR021/FS-4000/FS-4500/4700F)
  • 1st! 12×12 dot kanji (National FS-4600)
  • mouse and joystick (with option to use the keyboard as joystick)
  • Unique! JoyNet (experimental; connected to TCP/IP of host OS)
  • tape via CAS and WAV files (real recorded tape image); CAS files can be converted to WAV on the fly, cassette sounds are audible, cassettes can be rewinded and forced to play (like unplugging the 'remote control' plug on a real machine)
  • Unique! MSX RS-232 (8255 UART)
  • joystick emulation for mouse (a feature that the Philips SBC-3810 and Sony MOS-1 and similar mice have)
  • rensha turbo auto fire support
  • turboR pause (button) support

Emulator extras

  • Unique! XML files specify the hardware to be emulated, so you actually emulate specific MSX machines, which of course can be modified
  • various screen renderers available:
    • 640×480 hi-res renderer, with scanline emulation and support for 3 different scalers: hq2x, 2xSaI and scale2x (see animated images below)
    • 320×240 lo-res renderer (with compensation for lower resolution by blending)
    • Cool! 640×480 openGL based renderer that makes advantage of hardware acceleration of video cards and has some special effects: scanlines, blurring, afterglow (TV emulation, see also screenshots below), gamma adjustment
  • Unique! console with alpha-blended background and file and command completion
  • option to run at maximum possible speed
  • full screen rendering
  • (auto) frameskip
  • Unique! support for .xsa (compressed) diskimages
  • speed setting (in percent)
  • mouse cursor grabbing, for windowed mode
  • tunable volumes and channel modes for the individual sound chips, from the console
  • Unique! debug device: write data to special I/O ports and its get logged to stdout or a file in a format you specify. Very useful if you develop MSX assembly programs in openMSX
  • all read-only input files (ROM images, disk images, background pictures, XML) can be gzipped or zipped and will be transparently decompressed by openMSX. Note that some games require write access to their disk in order to run (for example, Seed of Dragon)
  • Unique! dir-as-disk feature: a directory can be specified instead of a disk image and the contents (up to 720K) will show up as files on the MSX. There is no support for subdirectories (as used by DOS2) yet. Note that this feature is still experimental
  • adjustable keymap (e.g. for Japanese PC keyboards or to use Russian MSX keyboards on a US English PC keyboard)
  • screen shot feature
  • Cool! debug command (including break points): read/write many parts of the MSX system; mainly useful to be used in external debug applications via the stdin command interface
  • Unique! keymatrixdown/up commands to automatically push keys (like pressing CTRL at boot automatically)
  • Unique! console commands to enable demo play with timers
