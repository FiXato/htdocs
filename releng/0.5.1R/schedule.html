openMSX 0.5.1 Release Process

Introduction

Here is a page for openMSX 0.5.1 release specific schedule. It is intended for the openMSX release engineering team and openMSX developers. End-users or for more generic information about openMSX and its development, see openMSX web site.

Currently, no general release engineering page of openMSX is available.

Most of detailed discussions are made on #openMSX at irc.freenode.net.

The current release engineering TODO list is also available.

Action Expected Actual Description Responsible Updated
Freeze features 2005-01-29 (UTC)   After this, no more new features can be commited. Even for critical bug fix, all commits must be approved by Release Engineer(s). Maarten, Wouter 2005-01-21 (UTC)
Freeze documents 2005-02-08 (UTC) 2005-02-25 At this point, all documents must be ready for release. Any description for all new features, all changed must be done. Version number in documents should be 0.5.1-Release. After this, even for critical fix, all commits must be approved by Release Engineer(s). Manuel 2005-02-25 (UTC)
Build 0.5.1-RC1 2005-02-07 (UTC)   Build 0.5.1-RC1 source tarball and win32 binary package and put on hidden space. This should be tested heavily by release engineers and core developers and core testers. Maarten, Herman 2005-01-21 (UTC)
Build 0.5.1-RC2     Build 0.5.1-RC2 and test. The needness of this is depend on the result of 0.5.1-RC1. So, no expected day. Maarten, Herman
